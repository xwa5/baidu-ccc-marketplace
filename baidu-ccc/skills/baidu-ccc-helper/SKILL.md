# 百度 Coding Config 助手

帮助用户管理和同步百度团队的 Claude 配置。

---

## ⚠️ 核心原则

**1. 使用 slash commands 而非直接执行：**
- 使用 `/baidu-ccc:install` 进行安装
- 使用 `/baidu-ccc:status` 查看状态
- 不要直接执行 `baidu-ccc-dev` 命令

**2. 用户体验：**
- 保持简洁自然的交互
- 不展示技术实现细节
- 根据用户意图选择合适的命令

## 何时使用

当用户提到以下场景时，主动使用此工具：

**安装/更新配置：**
- "安装团队配置"
- "同步 Claude 配置"
- "更新我的 coding config"
- 使用 `/baidu-ccc:install` 命令

**查看状态：**
- "查看配置状态"
- "配置是否是最新的"
- "团队有哪些共享的 commands/skills"
- 使用 `/baidu-ccc:status` 命令

## 可用命令

### 1. /baidu-ccc:install - 安装配置

安装或更新百度团队的 Coding Config 配置。

**流程：**
1. 自动检查并安装 `baidu-ccc-dev` 工具（如需要）
2. 启动交互式安装流程
3. 引导用户选择：
   - 配置级别（个人级/项目级）
   - 冲突处理方式（保留本地/使用远程）
4. 完成安装并展示结果

**特点：**
- 全自动化，无需手动输入命令
- 智能检测并处理配置冲突
- 清晰展示变更内容

### 2. /baidu-ccc:status - 查看状态

查看当前 Coding Config 的配置状态。

**显示内容：**
- 配置目录位置
- 配置来源 URL
- 最后同步时间
- 所有安装位置的状态（个人级、各项目级）
- 已安装的配置内容（skills、scripts、commands、hooks 等）

## 使用示例

### 示例 1：首次安装

**用户**: "帮我安装团队的 Claude 配置"

**你的操作**：
```
使用 /baidu-ccc:install 命令安装配置。

安装过程中会：
1. 选择配置级别（建议选择个人级）
2. 下载团队配置
3. 处理冲突（如有）
4. 完成安装
```

### 示例 2：检查更新

**用户**: "我的配置是最新的吗？"

**你的操作**：
```
使用 /baidu-ccc:status 查看当前状态。

如果显示配置较旧，建议使用 /baidu-ccc:install 更新。
```

### 示例 3：更新配置

**用户**: "更新一下我的 coding config"

**你的操作**：
```
使用 /baidu-ccc:install 更新配置。

更新过程会自动检测变更并处理冲突。
```

## 配置说明

### 配置级别

**个人级配置** (`~/.claude/`)
- 跨所有项目共享
- 建议用于通用的 commands、skills
- 只需安装一次

**项目级配置** (`./.claude/`)
- 仅当前项目使用
- 适合项目特定的配置
- 每个项目独立安装

### 冲突处理

当检测到本地和远程都有修改时：

1. **查看差异** - 显示具体变更内容
2. **保留本地版本** - 保持自定义修改
3. **使用远程版本** - 与团队配置保持一致

**建议策略：**
- 有自定义修改 → 先查看差异，再决定
- 没有修改过 → 直接使用远程版本
- 不确定 → 选择保留本地，稍后手动处理

## 配置内容

安装后会获得团队共享的配置，可能包括：

- **Skills** - 团队定制的技能工具
- **Commands** - 自定义的斜杠命令
- **Scripts** - 实用脚本工具
- **Hooks** - 各种钩子函数
- **Agents** - 专用代理配置

具体内容会随团队配置更新而变化，使用 `/baidu-ccc:status` 查看当前已安装的配置。

## 最佳实践

1. **定期同步**：建议每周运行一次 `/baidu-ccc:install` 获取最新配置
2. **优先使用个人级**：除非有特殊需求，建议安装到个人级配置
3. **查看状态**：定期使用 `/baidu-ccc:status` 检查配置是否最新
4. **冲突处理**：遇到冲突时先查看差异，避免覆盖重要的自定义配置

## 注意事项

- 配置是单向同步（从远程到本地）
- 如需贡献配置到团队，需通过其他方式（如 Git PR）
- 删除的文件和空目录会自动清理
- 安装的脚本会放到 `~/.local/bin/`，确保该目录在 PATH 中
