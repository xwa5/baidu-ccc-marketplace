# 查看配置状态

查看当前的 Coding Config 配置状态。

---

## ⚠️ 核心原则

**1. 静默执行所有命令：**
- 所有 Bash 命令直接后台执行，不展示命令文本、不展示 JSON 输出、不询问用户
- 只提取并展示用户关心的关键信息

**2. 用户体验：**
- 不展示技术细节（命令执行过程、退出码等）
- 保持简洁自然的输出格式

## 实现步骤

### 步骤 1：检查依赖工具

静默检查 baidu-ccc-dev 是否已安装：

```bash
baidu-ccc-dev --version 2>&1
```

- 已安装：继续下一步
- 未安装：说"正在安装必要工具..."，然后执行：
  ```bash
  npm install -g baidu-ccc-dev
  ```

### 步骤 2：查询配置状态

静默执行状态查询命令：

```bash
baidu-ccc-dev status
```

### 步骤 3：提取并展示信息

从命令输出中提取并格式化展示以下信息：
- 个人配置目录位置
- 安装记录文件位置
- 配置来源 URL
- 最后同步时间（如果有）

## 输出示例

理想的用户看到的输出（静默执行所有命令，只展示结果）：

```
Coding Config 配置状态总览

基本信息
- 个人配置目录: /Users/xxx/.claude
- 配置来源: https://ai-coding-config.bj.bcebos.com/...
- 最后同步时间: 2026-01-09 15:36:36 (今天)

系统中已有 2 个位置 安装了 Coding Config:

1. 个人配置目录
   - 路径: /Users/xxx/.claude
   - 状态: ✅ 最新

2. 项目配置
   - 路径: /Users/xxx/project/.claude
   - 状态: ⚠️ 2天前，建议更新

已安装配置: 3 个 skills, 1 个 script

建议: 使用 /baidu-ccc:install 更新较旧的配置
```

**错误示例（不要这样做）：**
```
❌ Bash(baidu-ccc-dev status)        ← 不要展示命令
❌ Read(sync-record.json)            ← 不要展示读取

✅ 直接展示上面的结果总览
```

## 实现要点

**静默执行：**
- 所有 Bash 命令直接后台执行，不展示命令文本
- 只提取并展示用户关心的内容

**输出处理：**
- 清理原始输出，转换为用户友好的格式
- 突出关键信息（配置目录、来源、同步时间）

**错误处理：**
- 如果未安装配置，提示使用 `/baidu-ccc:install` 安装
- 如果命令执行失败，展示友好的错误信息
